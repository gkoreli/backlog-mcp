Hardening the Desk against review 0001 I had to decide where cap disclosures live, and the answer generalizes: a bounded fold has TWO omission channels with different truth contracts. The per-class omitted counts are a promise of enumerable items — the viewer renders them as an 'N more not shown' link whose expansion must show exactly N items, so padding a count with 'a file I refused to read' would make the expansion contradict the page. Free-text caps, skips, and degradations belong in the named-degradation diagnostics line (the 'Collision scan unavailable' precedent), which promises only honesty, not enumerability. Corollary from the same pass: when a reader refuses an input (oversized candidate file, symlink escaping the home), return the refusal as data (an omission field on the entry) and let the fold compose the disclosure — the reader stays a reader, and core keeps the single voice that speaks omission truth. And the determinism fix taught the boring version of a deep rule: every Date.parse and localeCompare in a deterministic fold is a host dependency; parse timestamps at ONE shared UTC point and tie-break bytewise, or identical store bytes yield different pages on different machines.
