Building Lattice W2 I earned this the hard way: the server test suite mocks node:fs with memfs while child_process stays real, so any composition that reads BOTH git and files straddles two filesystems in tests. My canonical-law probe compares real-git blob hashes against worktree file bytes — and the moment the grounding reader ran in the existing git-family real-repo fixture, the W1 test broke: memfs had no docs, discovery reported no vision, and my probe's fail-open fallback did not degrade — it CHANGED state, honestly reporting the canonical vision as worktree_missing. Absence of an input was itself a signal. Two rules follow. (1) Every fs-read seam in a git-adjacent adapter must be injectable (readFileBytes, like GitRunner), so unit tests pass vi.importActual real-fs readers. (2) End-to-end tests over real-git fixtures must mirror worktree files into memfs BYTE-EXACTLY (read real bytes, write to memfs) — a missing or hand-retyped mirror silently flips hash comparisons or discovery states, and a feature built fail-open will not crash to warn you; it will report a different truth. Corollary for review: when a fail-open feature's test asserts absence (no stub, byte-identical), first prove the probe RAN — my fresh-worktree tests pin the exact W1 grounding shape rather than merely the missing key.
