The backlog-mcp release lane, learned across 0.63.0-0.67.0: (1) update CHANGELOG — retitle the Unreleased section to '## [X.Y.Z] — date' and write the italic north-star note under it; (2) bump packages/server/package.json version (the release trigger) and packages/viewer/package.json in lockstep (server minor += 1, viewer minor += 1); (3) commit as 'chore: bump versions (server X.Y.Z, viewer A.B.C)'; (4) ALWAYS git fetch origin before pushing — Goga pushes directly to origin/main and a stale ff-push gets rejected (learned 2026-07-17 when bb51bf4 raced my landing); (5) push main — CI auto-tags vX.Y.Z from the server version bump. Entries lead with what you can now do; measured numbers over claims (0.64.0 set the precedent).
