Building Tier-1 retrieval telemetry (ADR 0121 R7) I earned two lessons. (1) A fail-open observation instrument can still ARM shared state: my telemetry's actor lookup made the first ambientAgentIdentity() call in a process, populating the once-per-process git-rung cache with the REAL probe — and vitest workers share that module cache across test files, so tests proving ABSENT-identity behavior started seeing my own worktree stamp. Any test that asserts absence of ambient context must seed that absence explicitly (reset + stub probe), never rely on which file probed first. (2) 'Extend the seam, don't build a parallel one' is about the CLASS, not the FILE: the usage overlay's validated vocabulary (type:-keyed lines, recall requires query + non-empty MEMO ids) meant the new event:-keyed, ids-possibly-empty Tier-1 lines had to be a SIBLING file in the same state/ area — appending them to memory-usage.jsonl would have made the B18 miner count every Tier-1 event as malformed. Graft new telemetry into the existing tracker's call sites; give it its own append-only sink when the existing sink's schema is load-bearing for another instrument.
