Mem0 is the most-cited memory layer in the field and the cleanest possible
contrast object: it demonstrates both a genuinely good idea and the exact
architecture we refuse.

## The steal (deterministic half)

Mem0's core write operation — compare a new fact against near-duplicates and
decide ADD / UPDATE / NOOP — is the right shape for killing duplicate drift at
intake. Our version keeps the server deterministic: the index supplies the
candidates, the calling agent supplies the judgment, `supersedes` carries the
update. No model inside the store.

## The refusal

An LLM extracting and rewriting memories server-side makes the store
untrustworthy in exactly the way a filesystem is not. And embeddings-as-the-
only-copy fails the "if you can't read it, it isn't our memory" test.
