Kiro represents the spec-driven development wave: repo-resident markdown specs
treated as agent ground truth. Two facts make it permanent reference material.

## The mechanic worth stealing

Steering files carry `inclusion: always | fileMatch | manual | auto` in
frontmatter — conditional context loading expressed as a file convention.
That is exactly the grain of our progressive-disclosure system, and our
reproduction measured it against the real corpus before believing it: the
idea aligned, but neither coarse nor exact fileMatch patterns beat the
existing wakeup/search path at current corpus size. Parked with a precise
re-trigger, not rejected.

## The gap that positions us

SDD tools carry specs forward but carry no memory across changes — nothing
learns. A docs-native memory engine is the missing half, which is why this
cohort validates the category without competing in it.
