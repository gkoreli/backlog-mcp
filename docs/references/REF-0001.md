Orama is the in-process retrieval engine underneath every search-shaped verb in
the product: `backlog_search`, `recall`, and the hybrid pipeline that ranks
entities and memories. It indexes the declared searchable fields of every
substrate (including runtime-declared ones like this very document type) into
one schema with a flattened `search_text` lane, and serves BM25, vector, and
hybrid queries locally.

## Why it fits

- Embedded, dependency-free retrieval is what makes "local-first IS the
  architecture" real — search works offline, private, with no daemon.
- Hybrid BM25+vector matches the two-channel retrieval the memory loop needs
  (exact IDs and phrases via BM25; aboutness via vectors).

## Watch

Upstream is quiet (v3.1.18 since late 2024). The recorded contingency is an
engine-agnostic seam, not a rewrite — see the idea-garden entry and ADR 0116
R-2. Any engine change is fixture-gated against the sealed search baseline.
