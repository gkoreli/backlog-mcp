Letta is the MemGPT lineage grown into a platform: agents whose context is
composed of persistent memory blocks, with deeper tiers behind retrieval. Two
things earn it a permanent reference entry.

## Why it matters to us

1. **The MemFS pivot.** Letta Code moved agent memory into git-backed markdown
   files — frontmatter, an always-loaded `system/` area, the rest on demand.
   The largest funded competitor independently arrived at "the repo's files
   ARE the memory". Confirmation, not threat: our moat is substrates,
   lifecycle verbs, contradiction detection, and judged evaluation.
2. **Budgeted blocks.** Their memory-block size limits informed the wakeup
   budget discipline — reproduced on our real fixtures before adoption, where
   the invariant survived and the allocator did not.

## Where we part ways

Letta hosts agents; we refuse to (the store never orchestrates). Letta lets
LLMs rewrite memory server-side; our write path is deterministic and LLM work
happens in external agents against explicit tools.
