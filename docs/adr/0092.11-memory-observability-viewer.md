# 0092.11. Memory Observability in the Viewer

**Status**: Accepted — shipped 2026-06-11 (`88d81a8`). Verified: viewer typecheck clean, 96 tests green, production build passes. Visual spot-check in the live viewer: recommended next time the server runs
**Why now**: ADR 0092.10 closed the server side of the memory initiative;
the human-visibility half of the north star ("full visibility of memories,
context, artifacts… fully traceable") is the least-exposed part of it. No
surveyed competitor renders agent memory as navigable, auditable artifacts
(ADR 0092.5) — this is the differentiator, and today our own viewer shows
`entity_refs: [MEMO-0007, …]` as dead text.

## Grounding in the viewer code (audited 2026-06-11)

- `MetadataCard` (`components/metadata-card.ts`) is the single generic
  frontmatter renderer (ADR 0097: "MetadataCard renders every frontmatter
  key-value generically"). It already routes `{url,title}` references
  through the split pane — but **plain entity-id strings (`entity_refs`,
  `supersedes`, `parent_id`) render as inert text**. Evidence chains exist
  in the data and die in the UI.
- `SplitPaneState.openMcpResource(url)` is the existing navigation seam
  (ADR 0070); entity markdown lives at `mcp://backlog/tasks/<ID>.md`.
- The substrate registry gives the viewer memory fields for free (ADR
  0098); no memory-specific component exists or is needed.

## Design — improve the generic renderer, not a special case

1. **Entity-id linkification**: any string value (scalar or list item)
   matching the entity ID pattern (`isValidEntityId` from shared) renders
   as a link opening `mcp://backlog/tasks/<ID>.md` in the split pane.
   `entity_refs` (evidence chains, R-8), `supersedes` (correction lineage,
   R-1), `parent_id`, and bundle member ids all become navigable — for
   every entity type, not just memories. One change, whole-system win.
2. **Validity semantics**: a `valid_until` value in the past renders with
   an `expired` chip; `derived: true` renders as an `inference` chip
   (Hindsight's epistemic separation, made legible). Key-aware styling in
   the same renderer; no new component.

Deferred (next loops, in 0092.10's order): usage sparklines from the
JSONL; full contradiction surfacing (R-9 — needs a server detection
endpoint); agent-diary facet; demand-aware ripeness; corroboration boost +
capacity GC.
