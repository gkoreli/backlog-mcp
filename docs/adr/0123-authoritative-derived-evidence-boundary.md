# 0123 — The Authoritative / Derived / Evidence Boundary

## Context

The store's implicit taxonomy was two-class: docs are truth, everything
under the control dir is rebuildable. Running the destruction test honestly
("delete every derived artifact; reconstruct without losing meaning" —
PROMPT 0012 §7) exposes that as a lie. The operation journal and retrieval
telemetry are neither authoritative-for-reads nor reconstructible — they are
irreproducible *evidence*. And the usage overlay feeds recall ordering, so
byte-identical reconstruction after deleting it is impossible *by design*.

Three classes exist. This ADR names them, enumerates every on-disk artifact
into exactly one, and makes the boundary executable as a destruction gate.

## Decision — three classes, one gate

**R1 — Every artifact the system reads or writes belongs to exactly one of
three classes**, decided by three questions: *Is it truth for reads?* *Can
it be rebuilt from truth?* *Is it append-only history?*

- **AUTHORITATIVE** — truth for reads. Human-visible markdown under
  `docs/` plus committed home configuration. Never regenerated, never
  mutated by a read path (ADR 0117/0122 discipline). Loss is data loss.
- **DERIVED** — projections of authoritative state: index, caches,
  compiled registry state. Delete-safe and reconstructible: destroying any
  derived artifact and re-initializing must converge to the same serving
  behavior, modulo the declared evidence signals of R3. Never committed.
- **EVIDENCE** — irreproducible observation: the operation journal,
  retrieval telemetry, the usage overlay. Append-only; never truth for
  reads (it may be *read as history* — activity feeds, attribution,
  offline mining — but never as the source of entity content); its loss is
  honest history-loss, not corruption. Evidence writers fail open — they
  may never break a read or a mutation.

**R2 — The enumeration.** As of this ADR, exhaustively (paths relative to
one backlog home; project control dir `.backlog/`, global home `~/.backlog`
where the control dir is the root itself):

| Artifact | Class | Ground |
|---|---|---|
| `docs/**` (all substrate folders: `tasks/`, `memories/`, `adr/`, `prompts/`, `proposals/`, `references/`, `epics/`, …, plus root orientation docs) | AUTHORITATIVE | The corpus. `core/backlog-home.ts` (`BACKLOG_DOCUMENTS_DIR`) |
| `docs/substrates/*.json` (user-defined substrate definitions, ADR 0113) | AUTHORITATIVE | Law for interpreting the corpus |
| `.backlog/config.json` | AUTHORITATIVE | Committed home configuration (`core/config.ts`) |
| `.backlog/config.local.json` | AUTHORITATIVE (machine-local) | This machine's stated intent; uncommitted but not rebuildable — deleting it loses a decision, not a cache |
| `.backlog/.gitignore` | DERIVED (committed) | Tool-owned boundary marker; regenerated by `ensureProjectControlIgnores()` (`storage/local/local-runtime.ts`); committed only so git enforces this very boundary |
| `.backlog/cache/search-index.json` | DERIVED | Orama index + embeddings, stamped `INDEX_VERSION` (`@backlog-mcp/memory` `orama-schema.ts`); version mismatch or absence forces full rebuild from `docs/` (ADR 0101) |
| in-memory registry/resource state (compiled substrates, resource catalog) | DERIVED | Rebuilt every boot from `docs/` + definitions; listed for completeness — no file exists |
| `.backlog/state/operations.jsonl` | EVIDENCE | The operation journal (ADR 0094); read only as history (activity, attribution, per-task counts) |
| `.backlog/state/memory-usage.jsonl` | EVIDENCE | Usage overlay (ADR 0092.9); folds into the bounded recall multiplier — the one declared evidence→ordering channel |
| `.backlog/state/retrieval-telemetry.jsonl` (+ `.1` rotation) | EVIDENCE | Retrieval telemetry (ADR 0121 R7); never read in any serving path |
| `.backlog/state/legacy-internal/`, `.backlog/state/logs/legacy/` | EVIDENCE | Migration quarantine (`core/migrate-docs-native.ts`); preserved bytes of a past layout |

Git tracking states the same law: `docs/`, `.backlog/config.json`,
`.backlog/.gitignore` are committed; `cache/` and `state/` (the
`DERIVED_CONTROL_RULES`) and `config.local.json` never are.

**R3 — The destruction gate is a test, and it names what may differ.**
A structural server test (`__tests__/derived-destruction-gate.test.ts`)
builds a fixture home, indexes and searches it, deletes every DERIVED
artifact of R2 — never evidence, never docs — re-initializes, and asserts:

- the same entities are searchable (identical result sets, identical
  corpus-search ordering under fixed timestamps);
- structural properties hold (ids resolve, the rebuilt cache carries the
  current `INDEX_VERSION`);
- authoritative and evidence files are byte-identical through the
  destruction — the gate proves rebuild reads *only* truth and touches
  nothing else.

The single permitted difference is **usage-informed recall ordering**: the
overlay is evidence, survives derived destruction, and feeds the bounded
recall multiplier ("reorders, never hides", ADR 0092.9 R-15). If evidence
were also lost, recall *membership* must still be identical — only order
may move. The test asserts this by name. Anything else that differs after
destruction is corruption, and the gate fails.

**R4 — New artifacts must declare their class before they ship.** A PR
that adds an on-disk artifact under a home without extending the R2 table
is incomplete. An artifact that cannot answer the three questions cleanly
is a design smell (that is how this ADR was found: the usage overlay is
evidence that a derived consumer folds — the seam had to be named).

## What we refuse

- **No second source of truth.** Nothing derived or evidence-class is ever
  consulted to answer "what does this document say" — reconstruction reads
  `docs/` alone (ADR 0101 discipline; the journal is "positive evidence,
  not a completeness oracle", ADR 0119).
- **No committed derived state** beyond the self-enforcing
  `.backlog/.gitignore` marker.
- **No silent evidence loss**: deleting evidence is a legitimate operator
  act, but the system never does it implicitly (no rotation beyond the
  single declared telemetry rotation, no expiry, no compaction that drops
  lines).
- **No CI yaml** for the gate — it is a test in the suite, versioned with
  the code it constrains.

## Consequences

- Orama, the embedding model, and the viewer stay replaceable — enforced
  by a gate, not asserted in prose (pressure-map §7).
- Backup guidance falls out of the classes: commit authoritative, ignore
  derived, archive evidence if its history matters to you.
- The decay half of this boundary (pressure-map §5 Charter D1) ships
  beside this ADR: derived memories whose cited sources are superseded or
  terminal-closed gain a read-time `stale-source` marker on recall stubs
  and are counted in the Desk's REVIEW class — detection from existing
  relations at compose time, no stored flags, no write-time mutation, no
  automatic expiry, ranking untouched (ADR 0121 R5).
