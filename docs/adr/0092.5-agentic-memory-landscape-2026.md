# 0092.5. Agentic Memory Landscape 2026 — Research Findings & Design Rulings

**Status**: Accepted — research record + locked design guidance for Phases C–E
**Refreshes**: the ADR 0092 survey (2025-era; now ~14 months stale)
**Method**: primary-source review (repos, papers, vendor engineering blogs, third-party
benchmarks) of Hindsight, Mem0, MemPalace, Letta, Zep/Graphiti, and the 2025–26
memory-research literature, evaluated against our constraints: **local-first, no LLM
in the server's write path, memories as human-visible/editable markdown, one source
of truth**.

## TL;DR — three headline findings

1. **The field came to us on the write path.** Mem0 — the system ADR 0092 rejected
   for requiring an LLM on every write — publicly **abandoned its LLM-reconciled
   ADD/UPDATE/DELETE/NOOP pipeline** (Apr 2026): "reconciliation was slow, and it was
   where context got destroyed." Their replacement is ADD-only immutable records with
   retrieval-time temporal logic ([Mem0 blog](https://mem0.ai/blog/mem0-the-token-efficient-memory-algorithm)).
   Zep/Graphiti's bi-temporal model likewise *invalidates, never deletes*
   ([paper](https://arxiv.org/html/2501.13956v1)). Our deterministic, append-leaning,
   supersede-only design is now the field-validated position, not the contrarian one.
2. **Our echo problem has an architectural fix, not a statistical one.** Nobody in
   the field verifies that an agent actually *used* a memory — everyone proxies
   ("used" = retrieved, per Mem0's Memory Decay). The clean fix is **two-stage
   recall**: return stubs (id + title + digest) within a token budget; a follow-up
   expand call is a strong, deterministic, server-observable usage signal. This
   restores the echo signal ADR 0092.1 §Q4 lost when recall began returning full
   content — and cuts token cost as a side effect.
3. **LLM-free consolidation clustering is a solved sub-problem.** Entity bucketing
   (group unconsolidated episodics by dominant `entity_ref`/`state_key`) now;
   incremental label propagation over a deterministic memory graph later — Graphiti
   chose label propagation *specifically because it updates incrementally*
   ([repo](https://github.com/getzep/graphiti)). Recall-query demand is the ripeness
   signal. Every ingredient already exists in our data model.

## North star, sharpened

> An agent using backlog-mcp should be measurably smarter in week 10 than in
> week 1 — on the same project, because of memory. (ADR 0092.3)

This survey confirms the differentiator and sharpens it: every serious competitor
hides memory inside Postgres/vector stores behind an LLM write path; **none makes
the agent's memory a human-legible, human-editable, version-controlled artifact.**
The 2026 position statement:

> **backlog-mcp is the memory system you can read.** Evidence and inference are
> structurally separated, every derived belief traces to its sources, corrections
> are lineage (never overwrites), and the human can audit — and edit — what the
> agent knows, in markdown, in a live viewer, with git history.

## Per-system findings (distilled, with evidence)

### Hindsight (vectorize-io) — strongest new entrant

[Repo](https://github.com/vectorize-io/hindsight) · [paper, arXiv 2512.12818](https://arxiv.org/abs/2512.12818) · [consolidation post](https://hindsight.vectorize.io/blog/2026/05/21/agent-memory-consolidation)

- **Epistemic separation** is its core idea: four networks split *evidence* (World
  facts, first-person Experience) from *inference* (Opinions with confidence
  trajectories, per-entity Observations/"mental models"). Lesson for us: our layers
  capture half of this; a `derived` marker + mandatory provenance completes it.
- **Three timestamps per memory** (occurrence start/end + mention time) and
  **narrative facts over atomic fragments** (2–5 self-contained paragraphs per
  conversation; fragmentation measurably hurts retrieval — paper §4.1.2).
- Retrieval: four channels (vector, BM25, graph spreading activation, temporal) →
  RRF fusion (k=60) → **local cross-encoder rerank** (ms-marco-MiniLM-L-6-v2, no
  API) → **token-budget packing** (caller passes a budget, not top-k).
- **Observation trends** — stable/strengthening/weakening/stale, computed
  *deterministically* from evidence-arrival timestamps; mental models keep change
  history with diffs.
- Posture on forgetting: "eviction is a compliance tool, not a performance tool" —
  no native eviction; consolidation makes it unnecessary.
- Where it conflicts with us: the write path **requires** an LLM key by design;
  memory lives in Postgres, not human-editable artifacts. Benchmarks are strong
  (LongMemEval 91.4%, third-party leaderboard 94.6 — [agentmemorybenchmark.ai](https://agentmemorybenchmark.ai/)).

### Mem0 — the great reversal

[Token-efficient algorithm](https://mem0.ai/blog/mem0-the-token-efficient-memory-algorithm) · [temporal reasoning](https://mem0.ai/blog/the-token-efficient-memory-algorithm-now-has-temporal-reasoning) · [memory decay](https://mem0.ai/blog/introducing-memory-decay-in-mem0) · [v2→v3 migration](https://docs.mem0.ai/migration/oss-v2-to-v3)

- **ADD-only writes** (Apr 2026): contradictory facts coexist; retrieval-time
  temporal logic decides "what's true now". LoCoMo 71.4→91.6 from this change alone.
- **State keys**: memories about the same evolving fact share a key; a new state
  *closes* the old record with `event_end` — never deletes. **Memory kinds**
  (current / historical / plan / preference / timeless) assigned at write time.
- **Agent-generated facts became first-class** (their old pipeline largely ignored
  assistant output; fixing it took one LongMemEval category from 46.4 → 100.0) —
  direct validation of our implicit capture of agent task-completions.
- **Memory Decay** (May 2026): last-20 access timestamps → bounded 1.5×–0.3×
  rerank multiplier; never filters; candidate pool widens before truncation;
  reinforcement writes are async fire-and-forget. Attribution model is honest about
  its crudeness: "used" = retrieved.
- Graph memory (Neo4j) effectively deprecated in favor of an embedded **entity
  lookup layer** — vindicates our no-sidecar-DB tenet.
- Sobriety check: their own BEAM-10M numbers (temporal reasoning 16.3,
  contradiction resolution 32.5) show the field's hard problems remain open.

### MemPalace — the philosophical neighbor

[Repo](https://github.com/MemPalace/mempalace) · [memory stack](https://mempalaceofficial.com/concepts/memory-stack.html) · [contradiction detection](https://mempalaceofficial.com/concepts/contradiction-detection.html)

- Closest to our constraints: local-first, **verbatim storage, zero LLM in core**.
  96.6% R@5 retrieval-recall on LongMemEval with no LLM — proof the deterministic
  write path is benchmark-defensible. They publicly retracted earlier inflated
  claims and refuse dishonest cross-metric comparisons — a credibility posture
  worth copying.
- **L1 "essential story"** wake-up: top-15 importance items grouped by topic, hard
  char budget, *each line carries a source pointer* — a ready template for our
  wakeup knowledge section.
- **Closet layer**: summaries/pointers boost ranking but never gate; the verbatim
  drawer stays the source of truth — same shape as our digest+pointer capture.
- Battle scars worth inheriting: strict ISO-date validation at MCP boundaries
  (agents pass "March 2026"), write-time rejection of inverted validity intervals
  (silently-invisible-facts P0), kill-switch semantics (deleting the data dir
  disables capture; hooks never resurrect it).
- **Contradiction detection is still vaporware** (docs now banner it as planned,
  not shipped) — thirteen months after it was claimed. The lesson: don't put
  contradiction *resolution* in the write path at all (see Ruling 9).

### Letta, Zep/Graphiti, and the research frontier

- **Letta sleep-time compute** ([blog](https://www.letta.com/blog/sleep-time-compute)):
  every agent is secretly two — a primary and a background agent whose only job is
  rewriting memory into "learned context" during idle time. This is exactly our
  Phase D shape (external cron consolidator; server stays dumb), validated as a
  commercial architecture. Detail worth copying: the consolidator keeps *its own*
  memory (a cursor/watermark of what it has processed).
- **Zep/Graphiti** ([paper](https://arxiv.org/html/2501.13956v1)): bi-temporal
  edges (event time + ingestion time, `valid_at`/`invalid_at`); superseded facts
  invalidated never deleted; **every derived fact traces to a raw source episode**
  (the provenance invariant); incremental label propagation for community
  detection — the best LLM-free clustering primitive found.
- **A-MEM** ([arXiv 2502.12110](https://arxiv.org/abs/2502.12110)): the useful
  split — candidate generation is deterministic (embedding-kNN); only link/merge
  *judgment* uses an LLM. Maps 1:1 onto our server/agent boundary.
- **RL memory research** (Memory-R1, Mem-α, [Fine-Mem](https://arxiv.org/pdf/2601.08435)):
  consensus that end-task reward is too sparse — credit must anchor to **evidence
  actually cited**. The citation idea is liftable without RL (Ruling 11).
- **Eval landscape**: LongMemEval/LoCoMo near-saturated; BEAM (1M/10M) is the new
  hard benchmark; [agentmemorybenchmark.ai](https://agentmemorybenchmark.ai/) is an
  emerging third-party leaderboard.

## Design rulings (locked by this ADR)

Numbered so Phases C–E can cite them. Each grounded in the evidence above and our
four constraints.

**R-1. ADD-only, supersede-only.** `backlog_remember` only adds; `backlog_forget`
only soft-expires; `supersedes` is the *sole* correction mechanism; memory bodies
are never rewritten by tools. *(Mem0's reversal; Zep invalidation. Phase C.)*

**R-2. State keys.** Optional `state_key` frontmatter (e.g. `build.bundler`). A new
memory with an existing state key deterministically closes the predecessor
(`valid_until = now`, lineage recorded) — conflict handling with zero LLM, fully
greppable. *(Mem0 temporal reasoning. Phase C.)*

**R-3. Memory kinds + decay exemption.** Optional `kind` enum (current / historical
/ plan / preference / timeless) supplied by the *writing agent* (the LLM lives in
the agent, not the server). `timeless` — and by default the semantic/procedural
layers — are exempt from temporal decay; uniform decay is a bug both Mem0 and
Hindsight flag independently. *(Phases C+E.)*

**R-4. Occurrence time.** Optional `occurred_at` so a memory *about* an old event
isn't ranked fresh. Decay uses `occurred_at ?? created_at`. Full bi-temporality is
overkill for markdown. *(Hindsight three-timestamps; Zep. Phase C.)*

**R-5. Token-budget recall + two-stage expand.** `backlog_recall` returns **stubs**
(id, title, digest line, score) packed to a `token_budget`; `full: true` or a
follow-up `backlog_get(MEMO-id)` retrieves bodies. Expansion is the strong usage
signal (see R-11). *(Hindsight budget packing; MemPalace closets. Phase C — this
amends ADR 0092.3's "recall returns full content" stance, deliberately.)*

**R-6. Wakeup knowledge section = MemPalace L1 shape.** Top-N knowledge items
grouped by tag/entity, hard char budget, one source pointer per line. *(Phase C.)*

**R-7. Boundary validation + kill switch.** Strict ISO-date validation on
`valid_until`/`occurred_at` at the tool boundary with explicit errors; reject
inverted intervals at write time; deleting the data dir disables capture
permanently. *(MemPalace #1164/#1214/#1305. Phase C.)*

**R-8. Provenance invariant.** Every `derived: true` (consolidated) memory MUST
carry non-empty `entity_refs` to its source episodics — server-validated. The
viewer renders the evidence chain; consolidations become auditable and reversible.
*(Graphiti episodes; Hindsight evidence tracking. Phase D.)*

**R-9. Contradiction surfacing is a viewer feature, not a write-path feature.**
Deterministic detection (shared `state_key` with overlapping validity, or
near-duplicate embeddings with divergent content) flags pairs for **human
adjudication** in the viewer. Resolution is never automatic. *(MemPalace's
13-months-unshipped lesson; our HITL differentiator.)*

**R-10. Consolidation candidates: entity bucketing now, label propagation later.**
Server endpoint returns *bundles with evidence* — member ids, shared
entities/state keys, digest snippets, recall-demand stats — grouped by dominant
`entity_ref`. Ripeness = N≥k episodics per bucket within window W, near-duplicate
digest detection (cosine ≥ 0.9), and **frequent recall-query themes** (we already
log every recall). The external cron agent does pure synthesis, writing narrative
(not fragmented) semantic memories per R-8. *(Hindsight observations + A-MEM split
+ Graphiti clustering. Phase D.)*

**R-11. Composite usage signal, bounded influence.** Phase E usefulness =
f(**expand** count [strong], **citation** count [strong — MEMO-ids appearing in
subsequent server-visible writes, regex not honor-system], weak recall touches,
**corroboration trend** [new supporting episodics arriving via shared
entity/state_key]). Stored as last-N access *timestamps*, not a bare counter;
applied as a Mem0-style bounded multiplier (≈1.5×–0.3×) that reorders but never
hides; reinforcement writes async. *(Mem0 decay mechanics; Fine-Mem
evidence-anchoring; Hindsight trends. Phase E — supersedes ADR 0092.1 §Q4's
follow-up-call echo design.)*

**R-12. GC archives, never deletes.** Expiry/fizzle moves memories out of recall
(and eventually to an archive state), still greppable and viewer-visible. Hard
delete is a human-only command. *(Hindsight "eviction is a compliance tool";
ARTF-0189 tenet. Phase E.)*

**Rejected, with reasons:** LLM extraction in the server write path (constraint
violation; MemPalace proves 96.6% R@5 without it); LLM importance scoring at write
time (drifts across model versions); external graph DBs (Mem0 itself retreated);
summarize-then-drop compaction (universally panned in 2026 — "compaction is not
consolidation"); persona/disposition-conditioned opinion synthesis (belongs in
agents; epistemically opaque to humans).

## Best practices distilled (cross-system consensus)

- Evidence and inference must be structurally separated, and inference must cite
  evidence. (Hindsight, Graphiti)
- Never destructive: supersede/invalidate/expire; keep history; diffs are cheap
  when memories are files — git gives us belief history for free. (Mem0, Zep,
  Hindsight mental models)
- Usage signals bias ranking, never gate it; bound their influence; write them
  asynchronously. (Mem0)
- Narrative memories beat atomic fragments for both retrieval and human
  readability. (Hindsight §4.1.2)
- Determinism in the server, judgment in the agent — candidate generation,
  trends, clustering, validation are all computable without a model. (A-MEM,
  Letta sleep-time, Hindsight trend computation)
- Benchmark honestly: publish retrieval-recall numbers with reproduction scripts;
  refuse cross-metric comparisons. (MemPalace's retraction posture)

## Beyond the current roadmap (parked, not planned)

- **Memory templates**: preset frontmatter schemas + wakeup layouts per use-case
  (coding agent, support agent). (Hindsight Bank Templates)
- **Agent diaries as a viewer facet**: "what did agent X learn this week" — falls
  out of the existing `source` field nearly for free. (MemPalace)
- **Local cross-encoder rerank** stage for recall precision (~80MB, no API; adds
  read latency — optional flag). (Hindsight; ADR 0083 #9 remains deferred.)
- **Public honest benchmark page** for the zero-LLM pipeline on
  LongMemEval-retrieval/ConvoMem.
