# 0092.3. Agentic Memory — The Memory Experience

**Status**: Accepted — **Phases A & B shipped 2026-06-10** (see ADR 0092.4 for
the engineering record); Phases C–E pending. Note: the substrate shipped with
prefix **`MEMO-`**, not `MEM-` as sketched below — the codebase convention is
uniformly 4-char prefixes (TASK, FLDR, MLST, CRON) and the ID tooling assumes it.
**Target ADR**: 0092 (plugin-based agentic memory) — north-star re-grounding after Phases 1–3
**Supersedes in part**: ADR 0092.1 §Phases 4–5 sequencing; ADR 0092.2 §D4 (InMemoryStore as the episodic store)
**Defers**: cross-system memory sync, multi-user memory partitions, echo-signal redesign

## TL;DR

Phases 1–3 shipped the mechanics (decay, wakeup, capture, recall) but not the
*experience*. Memories die with the process, CLI recall is structurally always
empty, recall ranking is keyword matching while the real search pipeline sits
unused next door, and wakeup never touches the memory subsystem.

This ADR designs the memory layer from the agent's chair inward:

1. **The experience is four verbs** — `wakeup` (orient), `recall` (ask),
   `remember` (keep), `forget` (correct). Nothing else to learn. Implicit
   capture stays ambient; `backlog_get` already covers retrieval because of
   decision 3 below.
2. **A contract for "recall works properly"** — durable across processes and
   transports, ranked by the same hybrid pipeline as `backlog_search`, verified
   by a golden-query suite. `TASK-0676` / ADR 0083 move onto the critical path.
3. **Memory becomes a first-class substrate** (`MEM-` entities) and the default
   `MemoryStore` becomes a thin adapter over the backlog itself. This is the
   original 0092 thesis — "the backlog IS the memory" — taken to its
   conclusion, and it's what makes 1 and 2 cheap instead of a parallel stack.

## Part 1 — The experience we are designing

Design target: a day in the life of an agent session, week 10 of a project.

```
session start
│
├─ backlog_wakeup --scope FLDR-0001
│    → identity, active tasks, recent completions,        (exists today)
│      + top knowledge for this scope:                    (new)
│        "MEM-0107 [procedural] Release = typecheck → test → tag → publish"
│        "MEM-0093 [semantic]  Worker build must stay runtime-clean (ADR 0091)"
│
├─ ...work... hits a question past-self has answered
│   backlog_recall("how did we fix the OOM in the worker build?")
│    → MEM-0061 [episodic] "Fix build OOM — split tsc declarations from
│      tsdown bundling" (entity_refs: TASK-0588, ADR 0099) score 0.91
│   backlog_get("TASK-0588")          ← pointer hydration, existing tool
│
├─ ...completes a task → implicit capture, zero effort    (exists today)
│
├─ ...learns something durable
│   backlog_remember({ layer: "semantic",
│     content: "Goga prefers evidence bullets ≤1 line; no prose recaps.",
│     context: "FLDR-0001" })
│
└─ ...discovers stale knowledge
    backlog_remember({ supersedes: "MEM-0093", content: "..." })   ← correct
    backlog_forget({ id: "MEM-0042" })                             ← retract
```

### Experience principles

- **Four verbs, zero ceremony.** An agent should not need a memory discipline
  document. `wakeup` at start; `recall` when curious; `remember` when it
  learns; `forget` when it's wrong. Everything else is ambient.
- **Reading memory must be cheaper than re-deriving.** If recall is slow,
  wrong, or empty, agents (correctly) stop calling it. Recall correctness is
  therefore a contract with tests, not a hope — see Part 2.
- **Memory is content, pointers are hydration.** Recall returns the memory
  body in full (memories are small by construction). `entity_refs` link back
  to source entities; `backlog_get` hydrates them. No separate "retrieve"
  tool — and because memories are entities (Part 3), `backlog_get("MEM-0061")`
  works for free.
- **Correction is a first-class path.** Knowledge that can't be fixed rots.
  `remember({ supersedes })` is the update verb; `forget` is the delete verb;
  the human can edit any memory in the viewer via `write_resource`.
- **The human sees everything.** Memories render in the viewer like any
  entity. No surveyed system (MemPalace, Mem0, Karta — ADR 0092) gives the
  human a live, browsable, editable window into agent memory. We get it free.

### The tool surface (locked by this ADR)

#### `backlog_wakeup` — orient *(exists; gains a knowledge section)*

Unchanged shape, plus L2.5 — top semantic/procedural memories for the scope,
recalled through the composer. Wakeup becomes the read surface of memory
instead of a parallel query path. Budget: knowledge section ≤150 of the ~600
token target, capped at 5 entries, decay- and usage-ranked.

#### `backlog_recall` — ask *(exists; semantics tightened)*

```ts
backlog_recall({
  query: string,
  layers?: MemoryLayer[],     // default: all populated layers
  context?: string,           // scope to a container subtree (FLDR/EPIC/MLST)
  tags?: string[],
  limit?: number,             // default 5
  include_expired?: boolean,  // default false (Phase 5 semantics, durable field)
})
→ { items: [{ id, title, content, layer, context, tags,
              entity_refs, created_at, score }], total, query }
```

Changes vs today: default layer filter widens from `['episodic']` to all
populated layers (the agent asking "how do we deploy?" wants the procedural
answer, not just episodes); `context` becomes subtree-scoped (matches
`wakeup --scope` semantics) instead of exact-parent-match; results carry
`entity_refs` (plural — today's single `entity_id` generalizes).

#### `backlog_remember` — keep *(new)*

```ts
backlog_remember({
  content: string,            // markdown body — the memory itself
  layer?: 'episodic' | 'semantic' | 'procedural',  // default 'semantic'
  title?: string,             // default: first line of content, truncated
  context?: string,           // scope container id
  tags?: string[],
  entity_refs?: string[],     // source entities this knowledge derives from
  valid_until?: string,       // optional expiry (ISO date)
  supersedes?: string,        // MEM- id this replaces → old memory expires now
})
→ { id: "MEM-0042", ... }
```

The missing write-side for semantic/procedural knowledge. Implicit capture
(ADR 0092.2 rules — unchanged) keeps feeding `episodic`; `remember` is how
distilled knowledge enters. `supersedes` makes correction one call: the new
memory records lineage (`supersedes: MEM-0093` in frontmatter), the old one
gets `valid_until: now` — history preserved, recall stays clean.

#### `backlog_forget` — correct *(new; exposes the interface that exists since 0092)*

```ts
backlog_forget({
  id?: string,                // specific memory
  context?: string,           // or bulk: by scope
  layer?: MemoryLayer,        //          by layer
  older_than?: string,        //          by age
  expired?: boolean,          //          already-expired entries
})
→ { forgotten: number }
```

Forgetting is soft by default — `valid_until: now`, excluded from recall,
visible in the viewer with an expired badge. Hard deletion stays a human
action (`backlog_delete`), consistent with "agents mutate, humans steer."

#### Explicitly NOT in the surface

| Rejected tool | Why |
|---|---|
| `backlog_retrieve` | `backlog_get` on a `MEM-` id does this once memory is a substrate. |
| `backlog_memory_update` | `remember({ supersedes })` covers it with better lineage semantics. |
| A "store" distinct from "remember" | One write verb. Two names for writing is how tool surfaces rot. |
| Memory-specific search flags on `backlog_search` | Recall is the dedicated read surface; `search` excludes `type: memory` by default to keep the working corpus clean. |

## Part 2 — The recall-correctness contract

"Make sure recall works properly" is the second half of this ADR's mandate.
Today it provably doesn't:

1. **Durability hole.** The episodic store is `InMemoryStore`; the bootstrap
   composer is per-process. Capture (MCP server process) and recall (one-shot
   CLI process) never share a heap — `backlog-mcp recall` **cannot** return
   results. The flagship Phase 3 feature is unreachable from one of its two
   transports.
2. **Ranking hole.** `InMemoryStore.recall` scores by fraction-of-terms-found.
   No BM25, no embeddings, no compound tokenizer, no decay — while the hybrid
   pipeline (ADRs 0042, 0079–0081) lives in the same package.
3. **Inherited-bug hole.** ADR 0083 documents correctness bugs in that very
   pipeline (exact-ID matches buried, missing `await` in `indexResources`,
   MinMax normalization annihilating relevant low-BM25 docs — `TASK-0676`).
   ADR 0092.2 §D9 deferred Orama-backed memory until they're fixed. Routing
   recall through entity search makes them a hard blocker, on purpose.

The contract — recall "works properly" iff all five hold, each backed by a test:

| # | Property | Verification |
|---|---|---|
| R1 | **Durable**: memory captured in process A is recallable in process B (CLI↔MCP, across restarts) | Integration test: capture via core, new service instance, recall finds it |
| R2 | **Ranked**: recall rides the same `BM25 + vector → fusion → decay → coordination` pipeline as `backlog_search` | Golden-query suite for memory recall, modeled on `search-golden.test.ts` |
| R3 | **Exact**: recall by `MEM-` id, by referenced entity id ("TASK-0629"), or by distinctive title phrase surfaces the right memory at rank 1 | ADR 0083 failure cases, re-cut for memory corpus |
| R4 | **Scoped**: `context` filters to the container subtree; `layers`/`tags` filter natively (Orama `where`, ADR 0079) | Unit tests on filter composition |
| R5 | **Honest about time**: expired memories excluded by default; decay favors recent episodes but not at the cost of R3 exactness | Half-life tests + exactness-under-decay regression (the TASK-0676 lesson) |

## Part 3 — Memory as substrate (the means)

The cheapest way to satisfy Parts 1 and 2 is to stop maintaining a parallel
memory stack. **Memories become backlog entities.**

- New substrate declaration (ADR 0098 pattern — one declaration, everything
  derived): type `memory`, prefix `MEM-`, leaf. Frontmatter carries
  `layer`, `source`, `context`, `entity_refs`, `valid_until`, `usage_count`,
  `supersedes`. Body = the memory content.
- Default `MemoryStore` becomes **`BacklogMemoryStore`** — a thin adapter
  implementing the existing `store`/`recall`/`forget`/`size` interface over
  `IBacklogService` + the existing search service. The ADR 0092 plugin
  boundary is untouched (Mem0/MemPalace/custom stores stay pluggable);
  capture hooks in `core/create.ts`/`core/update.ts` don't change.
- `type: memory` is excluded from default `backlog_list`/`backlog_search`
  results; `backlog_recall` is the dedicated read surface.

What this buys, inherited rather than built: durability (markdown files,
same as every entity — fixes R1 by construction), hybrid recall ranking (R2),
viewer observability, operation-log attribution (ADR 0094), SSE reactivity,
D1/Worker parity (memory stops being Node-only), and durable homes for
`usage_count` (Phase 4) and `valid_until` (Phase 5) — both of which shrink to
small features instead of subsystems.

Costs accepted: shared ID space and index (mitigated by default exclusion);
capture write-amplification (~300-byte files on low-frequency events, bounded
by GC); `TASK-0676` becomes a hard blocker (honest — any real store inherits
it).

**Alternatives rejected:**

- *Finish Phases 4–5 on `InMemoryStore`* — builds the feedback loop on sand;
  usage counts on entries that evaporate measure nothing.
- *Dedicated `OramaEpisodicStore` + JSON sidecar* (the implied "Phase 3.1") —
  duplicates the whole storage stack (second index, second persistence, second
  reconciliation problem — ADR 0101 exists because drift is real, second D1
  story) and contradicts ADR 0097's one-markdown-substrate positioning: a
  binary sidecar is invisible to the viewer, the operation log, and
  `write_resource`.

## Decisions NOT revisited

- `MemoryStore` stays the plugin boundary (ADR 0092); `BacklogMemoryStore` is
  the new default implementation, not a new interface.
- No LLM on the implicit-capture write path (ADR 0092). Consolidation's LLM
  runs in an external agent against explicit tools.
- Capture rules — done-transition, artifact-create — unchanged (0092.2 §D6).
- The viewer stays read-only; memory edits flow through MCP tools (ARTF-0189).

## Success criteria

1. Kill the server, restart, `backlog-mcp recall "auth middleware"` from the
   CLI — returns the completion memory captured via MCP before the restart.
2. `backlog_recall("MEM-0061")` and `backlog_recall("TASK-0588 OOM fix")`
   both surface MEM-0061 at rank 1.
3. A memory captured via MCP is visible in the viewer within one SSE tick.
4. `backlog_wakeup --scope FLDR-0001` includes ≥1 semantic/procedural memory
   once Phase D has run; total briefing stays ≤800 tokens.
5. ADR 0083 golden queries stay green throughout — memory work cannot regress
   search, and vice versa.
