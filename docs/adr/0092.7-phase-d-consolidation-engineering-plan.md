# 0092.7. Agentic Memory Phase D — Consolidation: Engineering Plan

**Status**: Accepted — shipped 2026-06-10 (`8800d21`); engineering record: ADR 0092.8
**Implements**: ADR 0092.3 Phase D; ADR 0092.5 rulings R-8 (provenance
invariant) and R-10 (deterministic candidate bundling)
**Architecture tenet**: ADR 0097 — "the store doesn't act; actors act." The
server computes candidates deterministically; an **external agent** does the
synthesis. No LLM enters the server.

## What consolidation is

Episodic memories accumulate one per completion/artifact. They are history,
not knowledge. Consolidation distills clusters of related episodes into
**narrative semantic/procedural memories** — the "what we learned" layer that
wakeup surfaces and agents recall. This is the Letta sleep-time shape
(ADR 0092.5): a background consolidator agent, with the server as its dumb,
honest substrate.

## Grounding in the code (audited 2026-06-10)

- The operation log records **writes only** (`WRITE_TOOLS` in
  `operations/types.ts`); there is no recall-query log. The "recall demand"
  ripeness signal sketched in 0092.5 R-10 therefore has **no data source
  today** → deferred to Phase E, which builds usage tracking anyway. Phase D
  ripeness = deterministic count + age.
- Episodic memories already carry everything bucketing needs:
  `parent_id` (context), `entity_refs` (source pointers), `tags`
  (completion/artifact), `valid_until` (consolidated/retired = expired).
- `remember`/`forget` (Phase C) are sufficient consolidator verbs — writing
  the distilled memory and retiring its sources need no new write surface.
- The wakeup knowledge section (R-6) is the consumption surface — Phase D's
  output becomes visible there with zero extra work.

## Design

### D1 — `derived` marks inference; provenance is enforced (R-8)

New optional substrate field `derived: boolean`. A derived memory is
*inference* (consolidator output) as opposed to *evidence* (implicit capture,
direct observation) — Hindsight's epistemic separation, expressed in one
flag. **Server-enforced invariant**: `remember({ derived: true })` without
non-empty `entity_refs` is a `ValidationError`. Every conclusion must cite
its sources; the viewer can render the evidence chain; consolidations are
auditable and reversible.

### D2 — Candidate bundles are computed, not stored

New read-only core function `consolidationCandidates(service, params)`:

1. Load `type: memory`, filter to **live, non-derived episodic** memories.
2. Bucket deterministically — key precedence: `context` (parent_id), else
   first `entity_ref`, else `unscoped`.
3. Score ripeness: `ripe = count ≥ min_count AND oldest member age ≥
   min_age_days` (defaults 3 / 7).
4. Return bundles (ripe first, then by count) with: bucket key, context,
   member ids + digest lines (bounded), union of entity_refs, time span,
   ripeness, and echoed params.

Pure bucketing/ripeness functions live in `core/consolidation.ts` and are
unit-testable without a service. No new storage, no background process, no
state — recomputed on demand, consistent with "the store doesn't act."

### D3 — The consolidator is an external agent on a cron descriptor

A `cron` entity (ADR 0097 Extension 1) *describes* the recurring job; any
scheduler/agent runs it. The contract, embedded in the MCP tool description
so every consolidator sees it:

1. `backlog_consolidation_candidates` → take ripe bundles.
2. Per bundle: read members (stubs suffice; `backlog_get` for depth), write
   ONE narrative memory — `backlog_remember({ layer: semantic|procedural,
   derived: true, entity_refs: [member MEMO- ids + key source entities],
   context, kind })`. Narrative, not fragments (Hindsight §4.1.2).
3. Retire the members: `backlog_forget({ ids: members })` — soft-expire, so
   the episodes remain auditable and the derived memory's citations resolve.
4. The consolidator keeps its own watermark via a `state_key:
   consolidation.watermark.<scope>` memory (Letta's detail; zero server code).

Retired (expired) episodics drop out of future candidate computations
automatically — no "consolidated" bookkeeping field needed.

### Deferred (explicitly)

- Recall-demand ripeness + near-duplicate (embedding) detection → Phase E,
  alongside usage tracking.
- Label-propagation graph communities (0092.5 §4b) → when entity bucketing
  proves insufficient at real corpus sizes.
- Contradiction surfacing in the viewer (R-9) → viewer work, separate change.
