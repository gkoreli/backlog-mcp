# 0092.6. Agentic Memory — Phase C Engineering Record

**Status**: Accepted — Phase C of ADR 0092.3 shipped and manually validated 2026-06-10
**Implements**: ADR 0092.3 Phase C; ADR 0092.5 rulings R-1…R-7, R-12
**Commits**: `8276bec` (MemPalaceStore removal + store-returns-entry), `a977493` (ADR 0104 posture), `d40cc43` (Phase C), `3482ce5` (forget-dedupe fix)
**Verification**: 832 tests green (806 server + 26 memory), typecheck clean, plus a
manual end-to-end CLI validation against a real data dir (findings below)

## North star check-in

> An agent using backlog-mcp should be measurably smarter in week 10 than in
> week 1 — on the same project, because of memory. (ADR 0092.3)

After Phase C the **entire four-verb session loop from ADR 0092.3 Part 1 is
executable end-to-end**: wake up with knowledge, recall stubs, expand, work
(implicit capture), remember distilled knowledge with correction lineage,
forget what's wrong. What remains for the north star is intelligence on top
of a now-complete experience: consolidation (Phase D) and the feedback loop
(Phase E).

## What shipped

- **`backlog_remember`** (core + MCP + CLI) — the explicit knowledge write.
  ADD-only per R-1: `supersedes` soft-expires the predecessor and records
  lineage; `state_key` (R-2) deterministically closes every live holder of an
  evolving fact ("deploy.method") — conflict handling with zero LLM. Strict
  boundary validation per R-7: ISO dates with explicit, agent-readable errors;
  inverted validity intervals rejected at write time; id-shape checks on
  `context`/`entity_refs`/`supersedes`.
- **`backlog_forget`** (core + MCP + CLI) — soft-expire by ids / context /
  layer / older_than (`valid_until = now`; the record stays auditable in the
  viewer); `expired: true` is the hard-delete GC path (R-12). Hard deletion of
  live memories remains a human action.
- **Stub recall (R-5)** — `backlog_recall` now returns stubs (id + one-line
  digest) by default; `full: true` returns bodies; `token_budget` packs
  greedily (chars/4) with a `truncated` flag. This deliberately amends
  0092.2 §D8's full-content recall: the stub→expand round-trip is the
  deterministic usage signal Phase E's echo tracking will consume — and it
  makes recalls cheaper.
- **Wakeup knowledge section (R-6)** — L2.5 between "now" and "recent": top
  semantic/procedural memories for the scope, char-bounded lines, one source
  pointer each (MemPalace's L1 "essential story" shape). `--max-knowledge`
  caps it; 0 omits it.
- **Substrate fields (R-2/R-3/R-4)** — `state_key`, `kind`
  (current/historical/plan/preference/timeless), `occurred_at`. Decay rules:
  semantic/procedural layers and `kind: timeless` are **exempt** from temporal
  decay (uniform decay over stable knowledge is a bug — Mem0 and Hindsight
  both flag it); episodic decays on `occurred_at ?? created_at`, so a memory
  *about* an old event doesn't rank fresh.
- **Plugin interface evolution** — `MemoryStore.store` now returns the entry
  *as stored*: stores mint canonical ids (BacklogMemoryStore assigns MEMO-),
  and `remember` must report the real id. Composer signature updated.
- **`MemPalaceStore` removed** — the pythonia-bridge adapter from the 0092
  exploration era was untested, unused, and carried an unmet optional peer
  dependency. The `MemoryStore` interface remains the plugin boundary;
  adapters can live out of tree. (Per maintainer direction: competitors are
  for inspiration, not residence in the codebase.)
- **Architecture/posture captured** — ADR 0104: local-first is the primary
  mode; Workers+D1 is a maintained satellite, no feature-parity pressure.
  Engineering principles codified in AGENTS.md (composable modular code with
  JSDoc, no god files, composition over inheritance, core-first layering).

Phase C respects the core-first architecture throughout: `core/remember.ts`
and `core/forget.ts` are standalone, transport-free, individually documented
modules; the MCP tools and CLI commands are thin param-mapping adapters; any
future consumer reuses the same core.

## Manual validation — what we proved (and what it caught)

A full session loop was executed against a real data dir
(`BACKLOG_DATA_DIR=/tmp/memval`), with **every CLI command running as a
separate OS process** — the honest version of the R1 durability test:

- ✅ `remember` minted `MEMO-0001` with layer/kind/state_key/refs frontmatter;
  a **fresh process** recalled it by meaning ("how do we deploy" → stub with
  pointer). R1 holds across process boundaries for real, not just in tests.
- ✅ A second `remember` with the same `state_key deploy.method` closed
  `MEMO-0001` (`valid_until` stamped); recall then returned **only** the new
  holder. The closed memory's markdown file on disk is fully human-readable —
  frontmatter shows exactly what happened and when. The "memory system you
  can read" claim (ADR 0092.5) is observable in a `cat`.
- ✅ `remember --supersedes MEMO-0003` expired the predecessor and recorded
  lineage on the successor.
- ✅ `forget --expired` (GC) hard-deleted exactly the three expired memories
  and left live ones untouched.
- ✅ Implicit capture E2E: `create` → `update --status done --evidence …` →
  fresh-process `recall "SSE reconnect"` returned the completion memory with
  its `→ TASK-0001 [completion]` pointer. Bonus: the exact-title pin (Phase A)
  was visible in the score — the pipeline pieces compose.
- ✅ R-7 boundary errors are agent-legible:
  `occurred_at must be an ISO date or datetime (e.g. "2026-06-10" …); got "March 2026"`.
- ✅ `wakeup` rendered identity + the knowledge section with kind annotation
  and source pointer.
- 🐛 **Caught a real bug unit tests missed**: `forget --ids MEMO-0002`
  reported **3** forgotten. `BacklogMemoryStore` is registered for three
  layers; `composer.forget` invoked the same instance three times
  concurrently via `Promise.all`, racing it against itself — each call read
  the memory as unexpired and counted the soft-expire. Fixed by deduping
  store instances (recall already did this); regression test added
  (`composer.test.ts`). Lesson recorded below.

## Distilled insights

- **Manual validation earns its keep.** The forget triple-count was invisible
  to unit tests because they register *distinct* store instances per layer;
  only the production wiring (one instance, three layers) exposes the race.
  Contract tests now exist, but the rule stands: after each phase, run the
  real loop in real processes before writing the record.
- **The stub→expand amendment is the right call in practice.** Stub lines
  (`score id digest → pointer [kind]`) were comfortably scannable in the CLI
  and obviously cheaper; full bodies were one flag away. The usage signal this
  buys for Phase E came at zero experience cost.
- **Interface evolution beats interface workarounds.** `store()` returning
  the stored entry was a two-line change in each store; the alternative
  (query-after-write to learn the minted id) would have been racy and slow.
  When the plugin boundary is yours, evolve it deliberately.
- **Deterministic correction semantics feel good to use.** `state_key`
  closing previous holders is one flag at write time and produces a clean,
  greppable history — no reconciliation pass, no model call, exactly what the
  Mem0 retreat (ADR 0092.5) predicted.
- **Validation messages are agent UX.** MemPalace's "agents WILL pass
  'March 2026'" scar transferred verbatim — the explicit error format was hit
  on the first manual attempt at a bad date.

## Next phases

- **Phase D — consolidation** (per 0092.5 R-8/R-10): server endpoint
  `backlog_consolidation_candidates` returning evidence bundles (entity/
  state-key buckets, near-duplicate digests, recall-demand stats); a `cron`
  entity describes the job; an external agent synthesizes narrative semantic
  memories via `remember` (with mandatory `entity_refs` provenance — R-8) and
  retires episodics via `forget`. The consolidator keeps its own watermark
  memory (Letta's sleep-time detail).
- **Phase E — feedback loop** (R-11/R-12): expand-count (stub→`backlog_get`)
  + MEMO-id citations in subsequent writes + corroboration trend → bounded
  usage multiplier (1.5×–0.3×) stored as access timestamps in `usage_count`'s
  successor; GC archives rather than deletes.
- **Smaller follow-ups**: operation-log entries for remember/forget (memory
  writes are currently invisible to the activity panel); viewer affordances
  for memories (expired badge, supersedes chain rendering, contradiction
  surfacing per R-9); `include_expired` recall param.
