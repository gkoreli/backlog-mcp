# Step 0 Classification Audit — Task/task/makeTask/TaskStorage

**Mandatory pre-edit audit** for the merged 0106.2 + 0106.3 execution. Every hit of
`Task` / `task` / `makeTask` / `TaskStorage` classified as:

- **(a) genuine task substrate** → LEAVE
- **(b) generic-entity fossil** → RENAME
- **(c) projection / unrelated DTO** → LEAVE

Blind search/replace is forbidden. This document is the evidence the audit ran.

## Critical findings (gate the design)

### F1 — Zod `.strict()` rejects undefined-valued unknown keys (the if-ladder is load-bearing)
Empirically verified (zod 4.3.5):
`z.object({...}).strict().parse({ ..., schedule: undefined })` → **throws**
`Unrecognized key: "schedule"`. It is NOT lenient on `undefined`.

`core/create.ts:43` always passes `schedule/command/enabled` into the factory
(destructured from `CreateParams`, undefined for a task). The current
`if (x !== undefined)` ladder exists precisely to strip those undefined cross-type
keys before `.parse()`. **Deleting the ladder naively breaks task/epic/folder
creation.**

→ **Design decision**: replace the hand-maintained per-field ladder with a single
generic "drop undefined keys" pass over a union-derived input. This satisfies the
ADR intent (kill the hand-maintained field list + dead fields) while preserving
runtime safety. The ADR's literal "delete the if-ladder" is honored in spirit (no
per-field list remains); a one-line generic strip replaces it because `.strict()`
semantics require it. Documented as a deviation in the ADR.

### F2 — `total_tasks` / `total_epics` are a wire contract → LEAVE keys
Consumers: `server/hono-app.ts:219` (`taskCount: counts.total_tasks + total_epics`),
`tools/backlog-list.ts:18` (tool description text), `cli/commands/list.ts:12`,
`core/types.ts:96-97`, and ~8 test mocks. Per ADR 0106.3 §"Interface cleanup"
default → **keep the keys unchanged**. Only internal locals could be renamed; not
worth the churn. **No change to counts() keys.**

### F3 — test typecheck exclusion
`packages/server/tsconfig.json` excludes `src/**/*.test.ts`. Test type annotations
(e.g. `makeTask(): Task` in context-hydration) are validated only at vitest
runtime (types stripped). So a stale `Task` type ref in a test won't fail
`pnpm typecheck` — the full suite is the gate for tests.

### F4 — public re-export has zero external consumers
`server/src/index.ts:2` re-exports `CreateTaskInput`/`createTask`. Grep for any
importer of `'backlog-mcp'` package or `server/src/index` outside the file itself
= **zero hits**. Internal monorepo → rename outright (no deprecated alias needed).

## Blast-radius → alias migration path choice (0106.2 §2)
`type Task` from `@backlog-mcp/shared` is consumed broadly (search, snippets,
memory). Path **(a) replace** at definition would ripple wide. Chosen: keep `Task`
exported (its meaning *is* "task entity"), and **add** `TaskEntity` (+ siblings) as
the canonical narrowed aliases. `Task` remains a valid task-substrate-instance
alias per the glossary (it narrows to the task substrate). This is path (b)-lite:
additive, zero-risk, and the glossary already blesses `Task` = task instance.
