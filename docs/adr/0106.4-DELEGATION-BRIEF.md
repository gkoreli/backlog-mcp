# Delegation Brief — ADR 0106.4: Rename Entity Body `description` → `content`

**Task**: TASK-0690
**Spec (read fully first)**: `docs/adr/0106.4-entity-body-rename-description-to-content.md`

This is a **pure rename** of the entity body field. **No behavior change** other than
the field name (substrate, code, and the public MCP/CLI input field all move to `content`).

## MANDATORY Step 0 — classification audit (NO edits before this)

`description` appears ~262× in ~73 files and is **overloaded**. For every hit, classify:

- **(a) entity body → RENAME to `content`**: `BaseEntitySchema.description`, the memory
  substrate override, every `entity.description` / `x.description` property read where `x`
  is an Entity, every `description:` field in an **entity literal / schema**, storage
  seams, search-index field, `toMemoryEntry`.
- **(b) NOT the body → LEAVE**: `.describe('…')` calls; `description:` keys inside **Zod
  / JSON tool input schemas**; OAuth client/scope `description` (`auth/oauth-*.ts`);
  `package.json` description; any documentation string.

Write the audit to `docs/adr/0106.4-STEP0-AUDIT.md` before editing. Blind sed is
forbidden — it will corrupt population (b).

## Execution sequence

1. **Schema** (`packages/shared/src/substrates/`):
   - `base.ts`: `description: z.string().optional()` → `content: z.string().optional()`.
   - `memory.ts`: the override `description: z.string().min(1)` → `content: z.string().min(1)`
     (keep the doc comment; it already says "the body IS the content").
   - Check other substrates for any `description` override — rename if present.

2. **Storage seams**:
   - `storage/local/filesystem-storage.ts`: the `{ description, ...frontmatter }`
     destructure on write and `{ content: description } = matter(...)` read → use `content`.
     (Note `matter()` returns a `content` key for the body — alias carefully to avoid a
     name clash with the entity field; pick a clear local name.)
   - `storage/d1/d1-storage.ts`: **column stays `body`**. Rename only the `entity.description`
     side: `entity.description = row.body` → `entity.content = row.body`; `toNull(t.description)`
     → `toNull(t.content)`. Do NOT touch SQL column names or FTS `body` refs.

3. **Search** (`packages/memory/src/search/`): `orama-schema.ts`, `orama-search-service.ts`,
   `snippets.ts` — rename the entity-body field they index/read.

4. **Core + tools + CLI + context stages**: rename all entity-body `.description` reads and
   `description:` literals. Includes `core/edit.ts`, `core/search.ts`, `memory/capture.ts`,
   `memory/backlog-memory-store.ts` (`toMemoryEntry` → `content: m.content`), context stages,
   `resources/manager.ts`.

5. **Public MCP/CLI field** (ADR 0106.4 decision — rename to `content`):
   - `tools/backlog-create.ts` (+ any edit/update tool that takes a body): input-schema
     field `description` → `content`; update its `.describe()` text.
   - `cli/commands/create.ts`, `edit.ts`, etc.: flag/option `--description`/`description`
     → `content`; update help text.
   - Update README / docs examples that show the old field name.

Run the full suite between chunks.

## Leave untouched (population b — re-list)
- `.describe(...)` calls and tool-schema `description:` keys that describe a *parameter*,
  not the entity body.
- `auth/oauth-*.ts` `description`.
- `package.json` `description`.
- D1 SQL column `body` and `tasks_fts` `body` refs.
- On-disk `tasks/*.md` (positional body — do not migrate).

## Gate (definition of done)
- `pnpm -r typecheck` clean; full server suite green (885 baseline, 2 skipped); viewer green.
- Manually verify body round-trips: `backlog_create` with content → `get` shows it →
  `edit` updates it → `remember` + `recall` → `search` snippet. Throwaways deleted.
- `MemoryEntry.content` unchanged; `toMemoryEntry` no longer translates the body name.
- Reviewable commits per chunk; trailers `X-AI-Prompt` / `X-AI-Tool`.
- Update `docs/adr/README.md` to index 0106.4 + the Step-0 audit; mark ADR executed.
