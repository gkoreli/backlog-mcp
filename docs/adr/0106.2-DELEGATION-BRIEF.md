## MANDATORY Step 0 — classification audit (no edits before this)
For every hit of `Task` / `task` / `makeTask` / `TaskStorage` repo-wide, classify:
- **(a) genuine task substrate** (`TaskSchema`, `TaskSubstrate`, the task `z.infer`) → **LEAVE**
- **(b) generic-entity fossil** (`createTask`, `TaskStorage`, `task: Entity`, `allTasks`) → **RENAME**
- **(c) projection** (`*Item`, `*Result`) → **LEAVE**
Blind search/replace is forbidden — it corrupts genuine task-substrate refs.

## Execution sequence (merged)

**1. Factory cleanup (0106.2 §3)**
- `storage/schema.ts` → `storage/entity-factory.ts`
- `createTask` → `createEntity`; `CreateTaskInput` → `CreateEntityInput`
  derived from `z.input<typeof EntitySchema>` minus `created_at`/`updated_at`,
  `type` optional; **delete the if-ladder** + dead fields `due_date`/`content_type`/`path`.

**2. `<Type>Entity` aliases (0106.2 §2)** in `@backlog-mcp/shared`:
- `export type TaskEntity = Extract<Entity, { type: 'task' }>` (+ Epic/Folder/
  Artifact/Milestone/Cron/Memory). Migrate bare-`Task`-as-entity call sites.
  **Leave `TaskSchema`/`TaskSubstrate`.** Test helpers `makeTask(): Task` →
  `makeEntity(): TaskEntity`.

**3. Storage collapse + rename (0106.3 §A,§B)**
- **Delete** `storage/filesystem-adapter.ts` (dead pass-through wrapper, currently
  the class `FilesystemStorageAdapter`).
- Rename `TaskStorage` → **`FilesystemStorage`** (NOT `FilesystemStorageAdapter` —
  the pattern term lives on the interface; concrete class names the thing). It
  directly `implements StorageAdapter`. For symmetry rename `D1StorageAdapter` →
  `D1Storage`. **Interfaces keep `*Adapter`** (`StorageAdapter`/`AsyncStorageAdapter`).
- `BacklogService.storage: StorageAdapter = new FilesystemStorage()` —
  depend on the **interface**, not the concrete class. Preserve singleton + search
  bootstrap.
- Rename code symbols: `taskFilePath`→`entityFilePath`, `taskToMarkdown`→
  `entityToMarkdown`, `markdownToTask`→`markdownToEntity`, `iterateTasks`→
  `iterateEntities`, `task:` params → `entity:`.
- **GOTCHA**: keep the on-disk dir string constant `'tasks'` (rename only the
  symbol) — changing the value orphans existing data.

**4. Subfolder split + contract rename (0106.3 §C,§D)**
```
storage/
├── storage-adapter.ts            (interfaces)
├── backlog-service.contract.ts   (was service-types.ts — IBacklogService; ~40 importers)
├── entity-factory.ts             (was schema.ts)
├── local/{filesystem-storage.ts, backlog-service.ts}
└── d1/{d1-storage.ts, d1-backlog-service.ts}
```

**5. Remaining Task→Entity renames** across core/tests (0106.2 §1).

Run the full suite between chunks.
