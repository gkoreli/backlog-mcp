# 0119.1 — Implicit Identity Capture

## Context

ADR 0119 Slice A shipped attribution as an explicit parameter: `--as <agent>`
on every CLI write, an `as` field on every MCP write, or `BACKLOG_AGENT` in the
process environment. It works — memories render `by granite` — but the
ergonomics fail the signal-per-token test: the same identity fact is restated
on every call, and forgetting it once silently degrades attribution
(`identity: absent` in granite's own covenant wakeup). Goga's ruling
(PROMPT 0009, verbatim): identity must be captured implicitly.

The prior art is the tool we are already made of. Git never asks `--author`
per commit: identity is **workspace configuration** — declared once at the
scope it belongs to, resolved per operation through a precedence ladder, and
disclosed in the output. We adopt exactly that shape, including reusing git's
own config store, so agent identity needs no new file format, no new verb,
and no sync machinery.

## Decision — the ladder

Attribution for any write, and the briefing's `identity:` line, resolves
through the first present rung (most deliberate and most specific wins):

1. **Explicit per-call** — CLI `--as`, MCP `as` field. Rare override; unchanged.
2. **Worktree config** — `git config --worktree backlog.agent` (requires
   `extensions.worktreeConfig`; absent extension = rung absent). Stamped once
   when a delegation worktree is created; every call from inside that worktree
   is that agent's, with zero per-call cost.
3. **Environment** — `BACKLOG_AGENT`. The harness-session channel: Claude Code
   settings `env`, an `.mcp.json` server `env` block, a fleet spawn.
4. **Checkout config** — `git config --local backlog.agent`. For a checkout
   used by exactly one agent.
5. **User config** — `git config --global backlog.agent`. A human's or
   single-agent machine's standing default.
6. **Absent** — `identity: absent`, honest, exactly as today. Identity stays
   OPTIONAL and modular (PROMPT 0003); nothing is ever inferred from OS
   usernames or git `user.name`.

### Rulings

- **R1 — Worktree beats environment.** Environment variables are inherited
  indiscriminately by child processes: a builder spawned from granite's
  session carries `BACKLOG_AGENT=granite` through no choice of its own. The
  worktree stamp is placed deliberately, for that agent, at delegation time —
  the more deliberate signal must win or first-person attribution (PROMPT
  0006) breaks at the identity layer. This is the one place we deviate from
  git's own env-over-config precedence, and it is the point of the design.
- **R2 — Source is disclosed.** The briefing's meta line names the winning
  rung: `identity: granite (worktree config)` / `(env)` / `(--as)` /
  `(checkout config)` / `(user config)`. Misattribution must be debuggable at
  a glance, because implicit capture makes it silent by construction.
- **R3 — No new verbs, no new files.** `git config backlog.agent <value>` IS
  the interface, at every scope, exactly like git identity itself. The value
  is what `--as` accepts today: an `AGENT-` doc id or a declared `principal`.
  Resolution to a display title stays fail-open read-side (ADR 0119 R2):
  an undeclared value writes as the raw string and renders unchanged.
- **R4 — Shared checkouts are never stamped.** A checkout used by both a
  human and an agent (Goga's terminal and granite's session in the same
  directory) gets identity from the harness env (rung 3), which scopes to the
  session, not the directory. Stamping rung 4 in a shared checkout is the
  documented anti-pattern.
- **R5 — Delegation protocol, not product code.** Stamping
  `git config --worktree backlog.agent <builder>` at worktree creation is the
  orchestrator's covenant step (granite's delegation checklist), not
  something the engine does implicitly — the engine only reads.
- **R6 — Writers get worktrees (one writing actor per scope).** Goga's
  challenge: parallel agents or subagents in the same directory inherit the
  session env and would all claim its identity. Cases: worktree-isolated
  builders are covered (the R1 inversion exists precisely so the deliberate
  stamp beats the inherited env); read-only agents have zero exposure
  (attribution exists only on writes); a same-directory *writer* is the real
  hole. It is closed by protocol, not code: the merge law already requires
  writers to work in isolated worktrees — this extends that law from code
  writes to store writes. A subagent without a worktree is read-only; one
  worktree = one lease = one named authority (the lattice's lease model made
  real). Noted, not chartered: the journal already records the actor and W1
  computes worktree family — recording family per journal entry would make a
  stamp/claim disagreement auditable post-hoc. A tripwire, never a wall;
  build on trigger.

## Consequences

- Zero per-call identity tokens for every properly configured workspace; the
  flag survives for overrides only.
- Config reads ride the existing injectable git-runner seam (W1), fail-open on
  non-git homes and old gits — non-git behavior is byte-identical to today.
- Resolution is computed once per process (CLI) or per server boot and reused;
  no per-call git spawns on the hot path.
- The global home (`~/.backlog`) being a git repo participates naturally: a
  rung-4 stamp there attributes its direct CLI writes.

## Acceptance

- Precedence matrix test across all six rungs, including R1's
  worktree-over-env inversion and the worktreeConfig-extension-absent case.
- Source disclosure exact-text tests for the meta line.
- Non-git home: byte-identical wakeup meta versus pre-0119.1.
