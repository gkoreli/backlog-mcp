# 0092.8. Agentic Memory — Phase D Engineering Record

**Status**: Accepted — Phase D shipped and manually validated 2026-06-10

## North star check-in

With Phase D, the memory system **distills**. The 0092 capability table is
now four-for-five: decay ✅, wake-up ✅, implicit capture ✅, consolidation ✅,
echo/fizzle (Phase E) remaining. The semantic and procedural layers — empty
for the initiative's entire life until Phase C — now have a *production
pipeline filling them*: history becomes knowledge, knowledge surfaces in
wakeup, and every conclusion carries its evidence.

## What shipped

- **`core/consolidation.ts`** — pure `bucketEpisodics` (bucket key
  precedence: context → first entity_ref → unscoped; ripeness = count ≥
  min_count AND oldest member ≥ min_age_days) + read-only
  `consolidationCandidates(service, params)` over live, non-derived
  episodics. Bundles are **computed, never stored** — recomputed on demand,
  zero background state, per ADR 0097's "the store doesn't act."
- **`derived` flag** on the memory substrate — Hindsight's epistemic
  separation in one field: evidence (capture, observation) vs inference
  (consolidator output). **Server-enforced R-8 invariant** in
  `core/remember.ts`: `derived: true` without non-empty `entity_refs` is a
  `ValidationError` — a conclusion that cites nothing is unauditable and
  the server refuses to store it.
- **`backlog_consolidation_candidates`** (MCP) + **`consolidation-candidates`**
  (CLI). The consolidator contract ships *inside the tool description* —
  every agent that lists tools learns the workflow: take ripe bundles →
  write ONE narrative derived memory citing members → retire members via
  `backlog_forget` → keep a watermark via `state_key`.
- **Self-excluding retirement** — retired (soft-expired) episodics drop out
  of future candidate computations automatically. No "consolidated" flag, no
  bookkeeping migration; the existing expiry semantics carry the state.

Grounding finding from the plan (0092.7), confirmed in code: the operation
log records writes only, so the recall-demand ripeness signal has no data
source yet — deferred to Phase E where usage tracking creates one.

## Distilled insights

- **Expiry as state is a quiet superpower.** "Which episodics are already
  consolidated?" needed no schema, no flag, no migration — retirement
  (soft-expiry) already encodes it, and every other surface (recall, wakeup,
  size, GC) respects the same bit. Choosing soft-forget as the universal
  retraction in Phase C paid off one phase later.
- **Put the contract in the tool description.** The consolidator workflow
  lives where any MCP agent will actually read it — the tool listing — not
  in docs it will never fetch. The server can't run the agent, but it can
  teach it.
- **Enforced provenance changes the trust model.** With R-8 in the server,
  a human browsing the viewer can rely on an invariant, not a convention:
  if it's marked inference, the evidence links are there. This is the
  differentiator no surveyed system offers end-to-end (ADR 0092.5).
- **Deterministic bucketing was enough.** Context + entity_ref buckets
  produced exactly the right bundle on real captured data with ~10 lines of
  pure logic. Label propagation (0092.5 §4b) stays parked until a real
  corpus shows bucket quality breaking down.

## Next phase: E — the feedback loop (last of ADR 0092's five capabilities)

Per 0092.5 R-11/R-12, now unblocked by Phase C's stub recall and Phase D's
derived layer:

- **Usage signal**: count stub→expand (`backlog_get` on a MEMO- id recently
  returned by recall) and MEMO-id citations appearing in subsequent writes;
  store as last-N access timestamps; apply as a bounded (≈1.5×–0.3×)
  rerank multiplier that never hides results.
- **Corroboration trend**: new episodics sharing entity/state_key with a
  derived memory = it's alive; neither corroboration nor recalls = GC
  candidate.
- **GC**: archive (expire), never delete; hard delete stays human.
- This also creates the recall-demand log that Phase D deferred — closing
  the last open signal in the consolidation ripeness design.
