# Fix Home Layout: One `.backlog` Name, No Nested Control Dir

Two fixes to the home layout, before Phase E migration bakes it in.

**1. Rename `.backlog-mcp` → `.backlog`.** The `-mcp` suffix names the folder
after one wire protocol, but this thing is also a CLI, an HTTP server, and the
viewer. Name the folder after the thing (backlog), not the pipe (mcp) — the same
way it's `~/.claude` not `~/.claude-mcp`, `~/.aws` not `~/.aws-sdk`.

**2. Don't nest a control dir in the global home.** The control dir exists for
**projects only**: it's the gitignorable boundary next to committed `docs/` that
keeps the tool's scratch files out of the working tree. The global home has no
working tree to protect, so nesting is pure ceremony — the control dir there
*is* the root.

Result:

```
~/.backlog/          global: docs/ (truth) + cache/ state/ config.json inline
<repo>/.backlog/     project: gitignored control plane, next to docs/
```

Make `BACKLOG_CONTROL_DIR = '.backlog'` the project default, and have
`createGlobalHome` (`backlog-home.ts:149-159`) set `controlDir = root`.

This costs nothing architecturally: `BacklogHome` already parameterizes
`controlDir` (`backlog-home.ts:264-289`), the runtime reads `home.controlDir`
rather than hardcoding the name, and `isPathWithin` already allows
`controlDir === root` (`backlog-home.ts:251-259`). It's a constructor arg, not a
second code path — not the dual-format reader/writer R-11 forbids.

Amend ADR 0112 R-1/R-3: the docs/control split is universal, but the global home
collapses control into its root, and the control dir is `.backlog` (not
`.backlog-mcp`). The Phase E migration handles the on-disk `.backlog-mcp →
.backlog` rename for existing project users — settle now so it happens once,
since Phase E is already moving things.
