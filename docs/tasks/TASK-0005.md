Root cause: sequential id allocation is per-checkout with no cross-worktree coordination. This is a lattice-class problem (PROMPT 0003). Design space to explore BEFORE building: (a) merge-time renumber as documented protocol (current practice, works, manual); (b) allocation aware of family branches via W1 plumbing; (c) non-sequential ids for memories only. Needs an ADR ruling, not a quick patch — file under the lattice thread.
